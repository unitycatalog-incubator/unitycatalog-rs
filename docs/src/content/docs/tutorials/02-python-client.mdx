This tutorial shows how to install the `unitycatalog-client` Python package and use it to interact with a running Unity Catalog server. The Python client is built with [PyO3](https://pyo3.rs/), which compiles the Rust client library into a native Python extension.

## Prerequisites

- A **running Unity Catalog server** — follow the [Start Your First Server](/tutorials/01-first-server/) tutorial if you have not done so yet
- **Python 3.9+**
- **[uv](https://docs.astral.sh/uv/)** — Python package manager
- **Rust 1.85+** with `cargo` — required to compile the PyO3 extension
- **[just](https://just.systems/)** — task runner

## Step 1 — Build the Python wheel

From the root of the `unitycatalog-rs` repository, run:

```bash
just build-py
```

This compiles the Rust crate and produces a `.whl` file inside `python/client/`. The first build takes a few minutes because it compiles the full Rust dependency tree.

<Aside type="tip">
For iterative development you can install in editable mode instead, which avoids re-running `just build-py` after every Python-only change:

```bash
uv pip install -e python/client/
```

You still need to re-run `just build-py` (or `maturin develop` inside `python/client/`) whenever you change Rust code.
</Aside>

## Step 2 — Install the wheel

```bash
uv pip install python/client/
```

Verify the installation:

```bash
python -c "import unitycatalog_client; print('ok')"
```

## Step 3 — Connect to the server

Start a Python REPL or create a script. Import the client and point it at the local server:

```python
from unitycatalog_client import UnityCatalogClient

client = UnityCatalogClient(base_url="http://localhost:8080")
```

The `base_url` defaults to `http://localhost:8080`, so on a local setup you can also call `UnityCatalogClient()` without arguments.

## Step 4 — List catalogs

```python
catalogs = client.catalogs.list()
for catalog in catalogs:
    print(catalog.name)
```

If you followed the first tutorial, `my_catalog` should appear in the output.

## Step 5 — Create a schema

```python
schema = client.schemas.create(
    catalog_name="my_catalog",
    name="py_schema",
)
print(schema.full_name)  # my_catalog.py_schema
```

## Step 6 — Create a table

```python
from unitycatalog_client import ColumnInfo, TableType, DataSourceFormat

table = client.tables.create(
    catalog_name="my_catalog",
    schema_name="py_schema",
    name="py_table",
    table_type=TableType.EXTERNAL,
    data_source_format=DataSourceFormat.DELTA,
    columns=[],
)
print(table.full_name)  # my_catalog.py_schema.py_table
```

## Step 7 — Handle errors

When you request a resource that does not exist, the client raises a `NotFoundError`:

```python
from unitycatalog_client import NotFoundError

try:
    schema = client.schemas.get(
        full_name="my_catalog.does_not_exist",
    )
except NotFoundError as e:
    print(f"Schema not found: {e}")
```

Other common exception types follow a similar pattern — they map directly to HTTP 4xx/5xx responses from the server.

## Step 8 — Clean up resources

Delete the table, schema, and any other resources you created:

```python
client.tables.delete(full_name="my_catalog.py_schema.py_table")
client.schemas.delete(full_name="my_catalog.py_schema")
```

<Aside type="note">
Deletion order matters: you must delete tables before the schema that contains them, and schemas before the catalog that contains them.
</Aside>
