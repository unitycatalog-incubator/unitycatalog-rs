This tutorial walks you through starting the Unity Catalog server in in-memory mode and interacting with it via the REST API. By the end, you will have a running server with a catalog, schema, and table you created yourself.

## Prerequisites

- **Rust 1.85+** with `cargo` — install via [rustup](https://rustup.rs/)
- **[just](https://just.systems/)** — the task runner used throughout the project
- **curl** — for making HTTP requests (pre-installed on most systems)
- **git** — to clone the repository

Verify your Rust version:

```bash
rustc --version
# rustc 1.85.0 (...)
```

## Step 1 — Clone the repository

```bash
git clone https://github.com/unitycatalog-incubator/unitycatalog-rs
cd unitycatalog-rs
```

## Step 2 — Start the in-memory server

The `just rest` command compiles the server and starts it with an in-memory backend. Data is not persisted between restarts, which makes this mode ideal for exploration and development.

```bash
just rest
```

You should see output similar to:

```
Compiling unitycatalog-server ...
Finished release [optimized] target(s)
Listening on 0.0.0.0:8080
```

Leave this terminal running. Open a second terminal for the remaining steps.

<Aside type="note">
The server listens on `http://localhost:8080` by default. All API paths are prefixed with `/api/2.1/unity-catalog/`.
</Aside>

## Step 3 — Create a catalog

A catalog is the top-level namespace in Unity Catalog. Create one named `my_catalog`:

```bash
curl -s -X POST http://localhost:8080/api/2.1/unity-catalog/catalogs \
  -H "Content-Type: application/json" \
  -d '{"name": "my_catalog"}' | jq .
```

Expected response:

```json
{
  "name": "my_catalog",
  "comment": null,
  "properties": {},
  "created_at": 1700000000000,
  "updated_at": null,
  "id": "..."
}
```

## Step 4 — Create a schema

A schema lives inside a catalog and groups related tables together. Create `my_schema` inside `my_catalog`:

```bash
curl -s -X POST http://localhost:8080/api/2.1/unity-catalog/schemas \
  -H "Content-Type: application/json" \
  -d '{"catalog_name": "my_catalog", "name": "my_schema"}' | jq .
```

Expected response:

```json
{
  "name": "my_schema",
  "catalog_name": "my_catalog",
  "comment": null,
  "properties": {},
  "full_name": "my_catalog.my_schema",
  "created_at": 1700000000000,
  "updated_at": null,
  "schema_id": "..."
}
```

## Step 5 — Create a table

Tables belong to a schema. Create an external Delta table named `my_table`:

```bash
curl -s -X POST http://localhost:8080/api/2.1/unity-catalog/tables \
  -H "Content-Type: application/json" \
  -d '{
    "catalog_name": "my_catalog",
    "schema_name": "my_schema",
    "name": "my_table",
    "table_type": "EXTERNAL",
    "data_source_format": "DELTA",
    "columns": []
  }' | jq .
```

Expected response:

```json
{
  "name": "my_table",
  "catalog_name": "my_catalog",
  "schema_name": "my_schema",
  "table_type": "EXTERNAL",
  "data_source_format": "DELTA",
  "columns": [],
  "full_name": "my_catalog.my_schema.my_table",
  "created_at": 1700000000000,
  "updated_at": null,
  "table_id": "..."
}
```

## Step 6 — List resources to verify

### List catalogs

```bash
curl -s http://localhost:8080/api/2.1/unity-catalog/catalogs | jq .
```

```json
{
  "catalogs": [
    {
      "name": "my_catalog",
      ...
    }
  ],
  "next_page_token": null
}
```

### List schemas

```bash
curl -s "http://localhost:8080/api/2.1/unity-catalog/schemas?catalog_name=my_catalog" | jq .
```

### List tables

```bash
curl -s "http://localhost:8080/api/2.1/unity-catalog/tables?catalog_name=my_catalog&schema_name=my_schema" | jq .
```

## Step 7 — Shut down the server

Switch back to the terminal running `just rest` and press **Ctrl-C** to stop the server.

<Aside type="caution">
Because the in-memory server does not persist data, everything you created in this tutorial is lost when the server stops. See the [PostgreSQL-backed server tutorial](/tutorials/03-postgres-server/) to persist data across restarts.
</Aside>
