---
title: REST API Reference
description: All Unity Catalog REST endpoints with method, path, required permissions, and error codes.
---

All endpoints are mounted under the base path `/api/2.1/unity-catalog`. An interactive Swagger UI is available at `/api/2.1/unity-catalog/docs` when the server is running.

Each request is evaluated against the configured [Policy](/reference/trait-policy) using the `Principal` extracted from the request context. The required permission for each endpoint is listed in the tables below.

## Catalogs

| Method | Path | Permission | Description |
|--------|------|------------|-------------|
| `GET` | `/catalogs` | `Read` | List all catalogs |
| `POST` | `/catalogs` | `Create` | Create a new catalog |
| `GET` | `/catalogs/{name}` | `Read` | Get a catalog by name |
| `PATCH` | `/catalogs/{name}` | `Manage` | Update a catalog |
| `DELETE` | `/catalogs/{name}` | `Manage` | Delete a catalog |

## Error Codes

All error responses use the `application/json` content type and include an `errorCode` and `message` field.

| Error Code | HTTP Status | When It Occurs |
|------------|-------------|----------------|
| `RESOURCE_NOT_FOUND` | 404 | The requested resource does not exist |
| `RESOURCE_ALREADY_EXISTS` | 409 | A resource with that name or identifier already exists |
| `PERMISSION_DENIED` | 403 | The principal is not allowed to perform the action |
| `UNAUTHENTICATED` | 401 | No bearer token was provided, or the token is invalid |
| `INVALID_PARAMETER_VALUE` | 400 | A request parameter is malformed or missing |
| `INTERNAL_ERROR` | 500 | An unexpected server-side error occurred |

Example error body:

```json
{
  "errorCode": "RESOURCE_NOT_FOUND",
  "message": "The requested resource does not exist."
}
```
