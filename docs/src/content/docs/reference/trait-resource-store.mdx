The store layer is the persistence abstraction used by the server. All storage backends — in-memory, PostgreSQL, or custom — implement these traits.

<Aside>
The graph data model that motivates `ObjectLabel` and `AssociationLabel` is explained in [Graph Data Model](/explanation/graph-data-model).
</Aside>

## `ResourceStoreReader`

Read-only half of the store. All methods are `async` and return `Result<T>`.

```rust
#[async_trait]
pub trait ResourceStoreReader: Send + Sync + 'static { ... }
```

### Methods

#### `get`

```rust
async fn get(&self, id: &ResourceIdent) -> Result<(Resource, ResourceRef)>
```

Retrieves a single resource by its identifier. Returns the resource and a `ResourceRef` (typically the resolved UUID). Returns `Error::NotFound` if no matching resource exists.

#### `get_many`

```rust
async fn get_many(&self, ids: &[ResourceIdent]) -> Result<Vec<(Resource, ResourceRef)>>
```

Retrieves multiple resources by their identifiers. The default implementation calls `get` concurrently for each id using `try_join_all`. Implementations may override this for batched queries. Returns an error if any single lookup fails.

#### `list`

```rust
async fn list(
    &self,
    label: &ObjectLabel,
    namespace: Option<&ResourceName>,
    max_results: Option<usize>,
    page_token: Option<String>,
) -> Result<(Vec<Resource>, Option<String>)>
```

Lists resources of the given `ObjectLabel` type. When `namespace` is `Some`, only resources that are children of the named resource are returned. When `namespace` is `None`, the store root is used. Returns a page of results and an optional continuation token for the next page.

## `ResourceStore`

Read-write store. Extends `ResourceStoreReader`.

```rust
#[async_trait]
pub trait ResourceStore: ResourceStoreReader + Send + Sync + 'static { ... }
```

### Methods

#### `create`

```rust
async fn create(&self, resource: Resource) -> Result<(Resource, ResourceRef)>
```

Persists a new resource. The store generates and manages the `id` field; any `id` set on the input may be ignored. Returns the created resource with its assigned `ResourceRef`.

#### `delete`

```rust
async fn delete(&self, id: &ResourceIdent) -> Result<()>
```

Deletes a resource and all of its associations. Implementations must remove associations before removing the resource itself.

#### `update`

```rust
async fn update(
    &self,
    id: &ResourceIdent,
    resource: Resource,
) -> Result<(Resource, ResourceRef)>
```

Updates an existing resource identified by `id` with the provided `resource` value. Returns the updated resource.

#### `add_association`

```rust
async fn add_association(
    &self,
    from: &ResourceIdent,
    to: &ResourceIdent,
    label: &AssociationLabel,
    properties: Option<PropertyMap>,
) -> Result<()>
```

Adds a directed association edge from `from` to `to` with the given `label`. Associations are automatically made bi-directional: adding `A →[label]→ B` also stores `B →[inverse(label)]→ A`. Between two resources there can be at most one association with a given label. Returns `Error::AlreadyExists` if the association already exists.

#### `remove_association`

```rust
async fn remove_association(
    &self,
    from: &ResourceIdent,
    to: &ResourceIdent,
    label: &AssociationLabel,
) -> Result<()>
```

Removes the association from `from` to `to` with the given `label`. Implementations must also remove the inverse association. Returns `Error::NotFound` if the association does not exist.

#### `list_associations`

```rust
async fn list_associations(
    &self,
    resource: &ResourceIdent,
    label: &AssociationLabel,
    target_label: Option<&ResourceIdent>,
    max_results: Option<usize>,
    page_token: Option<String>,
) -> Result<(Vec<ResourceIdent>, Option<String>)>
```

Lists all associations of `resource` that have the given `label`. `target_label` optionally filters the associated resources by a specific identity. Returns a page of `ResourceIdent` values and an optional continuation token.

## `ProvidesResourceStore`

A marker trait for types that own a `ResourceStore`. Implementing this trait automatically derives both `ResourceStoreReader` and `ResourceStore` via blanket implementations.

```rust
pub trait ProvidesResourceStore: Send + Sync + 'static {
    fn store(&self) -> &dyn ResourceStore;
}
```

All calls to `ResourceStoreReader` and `ResourceStore` on a `ProvidesResourceStore` implementor delegate to `self.store()`.

## Blanket Implementations

Both `ResourceStoreReader` and `ResourceStore` are blanket-implemented for:

- `Arc<T>` where `T: ResourceStoreReader` / `T: ResourceStore` — delegates to `T`'s implementation, allowing stores to be freely shared behind `Arc`.
- Any `T: ProvidesResourceStore` — delegates every method through `self.store()`.

## `ObjectLabel`

Identifies the type of a resource node in the store's graph model. Derived from the `Resource` enum discriminants.

| Variant | Description |
|---------|-------------|
| `Share` | A Delta Sharing share |
| `Credential` | A storage credential |
| `Catalog` | A Unity Catalog catalog |
| `Schema` | A schema within a catalog |
| `Table` | A table within a schema |
| `Column` | A column within a table |
| `ExternalLocation` | An external storage location |
| `Recipient` | A Delta Sharing recipient |

`ObjectLabel` implements `Display`, `AsRefStr`, `EnumIter`, and `EnumString` (case-insensitive, serialized as `snake_case`).

## `AssociationLabel`

Labels directed edges between resource nodes. Every label has an inverse; when an association is stored, the inverse association is stored automatically.

| Variant | Inverse | Description |
|---------|---------|-------------|
| `OwnedBy` | `OwnerOf` | Resource is owned by another resource |
| `OwnerOf` | `OwnedBy` | Resource owns another resource |
| `DependsOn` | `DependencyOf` | Resource has a dependency on another |
| `DependencyOf` | `DependsOn` | Resource is a dependency of another |
| `ParentOf` | `ChildOf` | Resource is the parent of another |
| `ChildOf` | `ParentOf` | Resource is the child of another |
| `HasPart` | `PartOf` | Resource contains another as a part |
| `PartOf` | `HasPart` | Resource is contained within another |
| `References` | `ReferencedBy` | Resource references another |
| `ReferencedBy` | `References` | Resource is referenced by another |

`AssociationLabel` is serialized as `snake_case` JSON.
