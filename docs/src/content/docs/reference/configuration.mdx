---
title: Server Configuration Reference
description: Complete reference for Config, Backend, SecretBackend, and all configuration fields.
---

The server is configured via a JSON file passed on startup. All fields are optional; omitting them falls back to the defaults described below.

## `Config`

Top-level configuration object.

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `host` | `string \| null` | `null` (binds to `0.0.0.0`) | Host address to bind the server to |
| `port` | `number \| null` | `null` (uses `8080`) | TCP port to listen on |
| `backend` | [`Backend`](#backend) | `{ "engine": "in-memory" }` | Storage backend configuration |
| `secret_backend` | [`SecretBackend`](#secretbackend) \| `null` | `null` | Secret storage backend configuration |

## `Backend`

Selects the storage engine. Serialized as a tagged object with an `"engine"` discriminant field.

### `in-memory`

Stores all data in process memory. Data is lost when the server restarts. Intended for local development and testing only.

```json
{
  "backend": {
    "engine": "in-memory"
  }
}
```

Because `in-memory` is the default, an empty config object (`{}`) is equivalent.

### `postgres`

Persists data in a PostgreSQL database.

```json
{
  "backend": {
    "engine": "postgres",
    "host": "localhost",
    "port": "5432",
    "user": "uc_user",
    "password": { "env": "PG_PASSWORD" },
    "database": "unity_catalog"
  }
}
```

#### `PostgresBackendConfig` fields

| Field | Type | Description |
|-------|------|-------------|
| `host` | [`ConfigValue`](#configvalue) | Database server hostname |
| `port` | [`ConfigValue`](#configvalue) | Database server port (as a string) |
| `user` | [`ConfigValue`](#configvalue) | Database user |
| `password` | [`ConfigValue`](#configvalue) | Password for the database user |
| `database` | [`ConfigValue`](#configvalue) | Database name |

The resolved fields are assembled into a connection string of the form:

```
postgres://{user}:{password}@{host}:{port}/{database}
```

## `SecretBackend`

Optional backend for storing secrets (e.g. credential encryption keys). Serialized with an `"engine"` discriminant.

### `Postgres`

Stores secrets in the PostgreSQL database. Suitable for evaluation; for production workloads consider a dedicated secrets manager.

```json
{
  "secret_backend": {
    "engine": "Postgres",
    "encryption_key": { "env": "UC_ENCRYPTION_KEY" }
  }
}
```

#### `PostgresSecretConfig` fields

| Field | Type | Description |
|-------|------|-------------|
| `encryption_key` | [`ConfigValue`](#configvalue) | Key used to encrypt secrets at rest |

## `ConfigValue`

A leaf value that is either an inline string literal or a reference to an environment variable. Using an environment variable reference avoids placing credentials directly in the config file.

**Literal string:**

```json
"localhost"
```

**Environment variable reference:**

```json
{ "env": "ENV_VAR_NAME" }
```

If the referenced environment variable is not set, the field resolves to `None` and the server will fail to start or connect.

## Complete examples

### Minimal (in-memory, no secrets)

```json
{}
```

### PostgreSQL backend with mixed literal/env values

```json
{
  "host": "0.0.0.0",
  "port": 8080,
  "backend": {
    "engine": "postgres",
    "host": "db.internal",
    "port": "5432",
    "user": "uc",
    "password": { "env": "PG_PASSWORD" },
    "database": "unity_catalog"
  }
}
```

### PostgreSQL backend with secret storage

```json
{
  "host": "0.0.0.0",
  "port": 8080,
  "backend": {
    "engine": "postgres",
    "host": { "env": "PG_HOST" },
    "port": { "env": "PG_PORT" },
    "user": { "env": "PG_USER" },
    "password": { "env": "PG_PASSWORD" },
    "database": { "env": "PG_DATABASE" }
  },
  "secret_backend": {
    "engine": "Postgres",
    "encryption_key": { "env": "UC_ENCRYPTION_KEY" }
  }
}
```
